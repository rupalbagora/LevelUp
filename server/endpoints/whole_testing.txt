```
### 👤 Step 1 — Auth APIs

**1.1 Register User A**
```
POST /api/auth/register
Content-Type: application/json

{
  "username": "playerA",
  "email": "a@test.com",
  "password": "password123"
}
```
Expected: `201 { success: true, message: "User registered successfully" }`

**1.2 Register User B**
```
POST /api/auth/register

{ "username": "playerB", "email": "b@test.com", "password": "password123" }
```

**1.3 Login as User A** ← save the token!
```
POST /api/auth/login

{ "email": "a@test.com", "password": "password123" }
```
Expected: `200` with `user.token` in response. **Copy this token** → call it `TOKEN_A`.

**1.4 Login as User B** → save as `TOKEN_B`

**1.5 Check Auth**
```
GET /api/auth/check-auth
Authorization: Bearer TOKEN_A
```
Expected: `200 { success: true }`

**1.6 Logout**
```
POST /api/auth/logout

🌱 Step 2 — Seed Questions

```
### ⚔️ Step 3 — Battle Flow (Core)

**3.1 Create Battle (as User A)**
```
POST /api/battle/create
Authorization: Bearer TOKEN_A

{ "topic": "Arrays", "difficulty": "Easy" }
```
Expected: `201 { battleId: "..." }` → **Save this as `BATTLE_ID`**

**3.2 Try joining your own battle (should fail)**
```
POST /api/battle/join/BATTLE_ID
Authorization: Bearer TOKEN_A
```
Expected: `400 { message: "Cannot join your own battle" }`

**3.3 Join Battle (as User B)**
```
POST /api/battle/join/BATTLE_ID
Authorization: Bearer TOKEN_B
```
Expected: `200 { message: "Joined battle successfully", questionId: "..." }`

**3.4 Try joining again (race condition test)**
```
POST /api/battle/join/BATTLE_ID
Authorization: Bearer TOKEN_B   ← or any other user
```
Expected: `400 { message: "Battle already joined by another user" }`

**3.5 Get Question (as User A)**
```
GET /api/battle/BATTLE_ID/question
Authorization: Bearer TOKEN_A
```
Expected: `200` with question object (no hidden test cases exposed ✅)

**3.6 Get Question (as unauthorized user — should fail)**
Register a 3rd user, login, then:
```
GET /api/battle/BATTLE_ID/question
Authorization: Bearer TOKEN_C
```
Expected: `403 { message: "Access denied" }`

### 🏃 Step 4 — Run Code

**4.1 Run correct code (Python)**
```
POST /api/battle/BATTLE_ID/run
Authorization: Bearer TOKEN_A

{
  "language": "python",
  "code": "import sys\narr = eval(input())\nprint(max(arr))"
}
Expected: 200 { summary: "1/1 test cases passed", testResults: [...] }
4.2 Run wrong code
json{ "language": "python", "code": "print('wrong')" }
Expected: 200 but passed: false in results.
4.3 Run with unsupported language
json{ "language": "cobol", "code": "..." }
```
Expected: `500` (the `getVersion` throws — you may want to catch this more gracefully with a `400`)

### 📤 Step 5 — Submit Code

**5.1 Submit correct solution (User A)**
```
POST /api/battle/BATTLE_ID/submit
Authorization: Bearer TOKEN_A

{
  "language": "python",
  "code": "import sys\narr = eval(input())\nprint(max(arr))"
}
```
Expected: `201 { message: "Accepted", isCorrect: true, extendedEndTime: "..." }`

> After this, the 2-minute timer extension kicks in — `extendedEndTime` should appear.

**5.2 Try submitting again (duplicate check)**
```
POST /api/battle/BATTLE_ID/submit
Authorization: Bearer TOKEN_A  ← same user
```
Expected: `400 { message: "You have already submitted code for this battle" }`

**5.3 Submit wrong solution (User B)**
```
POST /api/battle/BATTLE_ID/submit
Authorization: Bearer TOKEN_B

{ "language": "python", "code": "print('nope')" }
```
Expected: `201 { message: "Wrong Answer", isCorrect: false }`

> Since both players have now submitted (`totalCount >= 2`), `completeBattle` should trigger automatically.
```

⚡ Step 7 — Socket.IO (Real-time)
Use this quick HTML file to test WebSocket events:
html<!DOCTYPE html>
<html>
<head><script src="https://cdn.socket.io/4.7.2/socket.io.min.js"></script></head>
<body>
<script>
  const BATTLE_ID = "PASTE_YOUR_BATTLE_ID_HERE";
  const socket = io("http://localhost:5000");

  socket.on("connect", () => {
    console.log("Connected:", socket.id);
    socket.emit("joinBattleRoom", BATTLE_ID);
  });

  // Listen for all key events
  ["battleStarted", "timerSync", "extraTime", "submissionUpdate", 
   "battleEnded", "userRunningCode"].forEach(event => {
    socket.on(event, data => console.log(`📡 ${event}:`, data));
  });
</script>
</body>
</html>
Open this in a browser, then perform the join/submit steps above — you should see events logged in the console.

Fix 2 — Serve the HTML file via a local server
Don't open test.html by double-clicking — that gives it file:// origin which browsers block. Instead:
bash# Install live-server globally (one time)
npm install -g live-server

# Run from your server folder
cd D:/Projects/LevelUp/server
live-server --port=5500
Then open http://localhost:5500/test.html in your browser.
